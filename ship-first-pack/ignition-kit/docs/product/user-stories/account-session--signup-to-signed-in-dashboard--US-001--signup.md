# User Story: Sign up and land signed in

## Parent Scope Slice

[Signup to signed-in dashboard](../scope-slices/account-session--signup-to-signed-in-dashboard.md)

## Status

`ready-for-spec`

## Story

As a founder, I create an account and reach a signed-in home screen, so that I can start using the product.

## Acceptance Criteria

- Given I am logged out, when I complete signup with valid credentials, then I am signed in without a second login step.
- Given I am signed in, when I open the app root, then I see a dashboard shell (not the marketing/signup page).
